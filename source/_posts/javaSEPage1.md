# 绪论

## 简介

&emsp;&emsp;Java自面世以来，凭借易学易用、功能强大的特点得到了广泛的应用。作为一门面向对象、能够跨越多平台、可移植性高的编程语言，Java不仅吸收理论C++语言的各种优点，还摒弃了C++里难以理解的多继承、指针等概念，作为静态面向对象编程语言的代表，极好地实现了面向对象理论，允许程序员以优雅的思维方式进行复杂的编程。

&emsp;&emsp;Java是介于编译型语言和解释型语言之间，可以代码编译成一种“字节码”，类似于抽象的CPU指令，然后针对不同平台编写虚拟机，不同平台的虚拟机负责加载字节码并执行，这样就实现了“一次编写，到处运行”的效果。

+ Java是一门用于开发应用软件的编程语言。
+ Java应用程序的入口方法是main( )
+ 在Java语言中，最基本的元素是对象
+ 运行Jar文件中class文件需要在Java命令后面加的参数为-cp
+ Java程序的执行过程中用到一套JDK工具，其中Javac.exe指Java语言编辑器
+ Java语言具有内存垃圾自动收集功能


## Java的诞生

+ 诞生于1995年，来自于Sun公司的一个称为Green的项目
+ 主要贡献者：James Gosling（Java之父）
+ 最初目的：为家电开发一个分布式系统，以便对其集中远程控制（由于C语言总是针对特定芯片，无法满足这一愿望）

&emsp;&emsp;Java是1995年6月由Sun公司推出的一门编程语言。1990年Sun公司成立了由James Gosling领导的开发小组，开始致力于开发一种可移植的、跨平台的语言，该语言能生成正确运行于各种操作系统、各种CPU芯片上的代码。他们的精心钻研和努力促成了Java语言的诞生。Java的快速发展得益于Internet和Web的出现，Internet上有各种不同的计算机，它们可能使用完全不同的操作系统和CPU芯片，但仍希望运行相同的程序，Java的出现标志着真正的分布式系统的到来。

&emsp;&emsp;Gosling将Java语言最初命名为oak，但发现与另一门编程语言重名，当Gosling与同事在咖啡馆喝着来自爪哇岛（Java）的咖啡的时候，有人就建议用Java这个名字看，于是得到大家的赞赏，Java这个名字就传开了。

## Java技术平台

&emsp;&emsp;针对不同的开发市场，SUN公司将Java划分为三个技术平台，它们分别是JavaSE、JavaEE和JavaME。

&emsp;&emsp;按照应用范围，Java可以分为3个体系，即Java SE、Java EE、Java ME。下面简单介绍一下这三个体系。

### Java SE	

&emsp;&emsp;Java SE（Java Platform Standard Edition，Java 平台标准版）是Java技术的核心，提供基础的Java开发工具，执行环境与应用程序接口(API),并为Java EE提供基础，主要应用于桌面应用程序的开发。

&emsp;&emsp;Java SE（Java Platform Standard Edition）标准版，是为开发普通桌面和商务应用程序提供的解决方案。JavaSE是三个平台中最核心的部分，JavaEE和JavaME都是从JavaSE的基础上发展而来的，JavaSE平台中包括了Java最核心的类库，如集合、IO、数据库连接以及网络编程等。

### Java EE	

&emsp;&emsp;Java EE（Java Platform Enterprise Edition，Java 平台企业版）它提供 Web 服务、组件模型、管理和通信API接口，主要用于网络程序和企业级应用的开发。但需要强调的是，任何Java语言学习者都要从Java SE开始，Java SE是Java语言的核心，而Java EE是在其基础上扩展的。

&emsp;&emsp;Java EE(Java Platform Enterprise Edition) 企业版，是为开发企业级应用程序提供的解决方案。Java EE可以被看作一个技术平台，该平台用于开发、装配以及部署企业级应用程序，主要包括Servlet、JSP、JavaBean、JDBC、EJB、Web Service等技术。

### Java ME	

&emsp;&emsp;Java ME（Java Platform Micro Edition，Java平台微型版）以前称为J2ME，也叫K-JAVA。Java ME为在移动设备和嵌入式设备（比如手机、PDA、电视机顶盒和打印机）上运行的应用程序提供一个健壮且灵活的环境。

&emsp;&emsp;Java ME(Java Platform Micro Edition) 小型版，是为开发电子消费产品和嵌入式设备提供的解决方案。JavaME主要用于小型数字电子设备上软件程序的开发。例如，为家用电器增加智能化控制和联网功能，为手机增加新的游戏和通讯录管理功能。此外，Java ME还提供了HTTP等高级Internet协议，使移动电话能以Client/Server方式直接访问Internet的全部信息，提供高效率的无线交流。

> :bulb: 提示：Java的3个体系的区别
> &emsp;&emsp;Java可以分为3个体系，即Java SE、Java EE、Java ME主要是面向的应用领域不一样。Java SE主要做一般的Java应用，比如腾讯QQ应用软件等。Java EE即J2EE，主要做企业应用，比如公司网站、企业解决方案。Java ME主要面向嵌入式等设备应用的开发，比如手机游戏等。

## Java的现状

&emsp;&emsp;Java共有3个体系，JavaSE（标准版本）、JavaEE（企业级应用版本）和JavaME（移动端版本）。
+ JavaEE多用于企业级项目开发，主要包括电信、互联网、物流和大型网站等。
+ JavaME广泛应用于小型设备尤其是移动设备上的开发，包括手机、PDA、机顶盒、汽车通信设备等。
+ JavaSE程序包括部署在桌面程序和那些为JavaWeb服务开发的Plain Java类。JavaSE是JavaEE和JavaME的基础。

## Java的特点

### 简单

&emsp;&emsp;Java放弃了C/C++中指针的使用，使编程简化、错误率大大降低。Java的内存管理是由系统自动完成的。

### 平台无关性

&emsp;&emsp;不管运行环境是Windows 、Unix还是Linux，Java可以在任何环境中运行，只要这个环境中安装了Java虚拟机和解释器。

> :memo: Java语言具有“一旦写成，处处可用”（write once，run anywhere）的平台无关性，是因为Java源程序被编译之后生成的.class字节码文件在各种操作系统下都是通用的，都可以被安装在特定操作系统的Java解释器正确地解释运行。

### 面向对象

&emsp;&emsp;面向对象的三个特性：封装、继承和多态。

&emsp;&emsp;面向对象的开发方式与人的思维方式更加接近，减少了程序员在“程序域”和“问题域”之间的翻译难度。

&emsp;&emsp;C++也是面向对象的，但由于要和C语言兼容，它保留很多面向过程的成分。

&emsp;&emsp;相比C++，Java语言是纯粹的面向对象的语言，也是讲授面向对象理念的理想语言。

### 健壮

&emsp;&emsp;Java在数组访问之前先判断是否越界，这减少了很多运行时的不确定性。

&emsp;&emsp;Java的类型检查更加严格，可以让编译器预先发现更多的错误。

&emsp;&emsp;Java不允许程序员操纵内存，而是由系统自己控制内存，这样就减少了内存出错的可能性。

### 分布式

&emsp;&emsp;Java实现了数据的分布，使得系统所操作的数据可以分散存储在不同的网络节点上，ava实现了操作的分布，即应用系统的计算可以由不同的网络节点完成。

&emsp;&emsp;Java应用程序可以打开并访问网络上的对象，其访问方式与访问本地文件系统几乎完全相同。

### 安全

&emsp;&emsp;Java只支持引用，不支持指针操作，这样可以很有效地抑制恶意程序的威胁。

&emsp;&emsp;内存的开辟和释放等工作交由系统自动完成，可以降低内存泄漏等很多安全风险。

&emsp;&emsp;Java语言在执行很多操作前，都要进行多次校验，确保不存在伪造的引用，没有违反权限控制的访问，访问对象时也要严格遵循对象的规范。这些都大大提高了Java程序的安全性。

1.简单性

&emsp;&emsp;Java语言是一种相对简单的编程语言，它通过提供最基本的方法完成指定的任务。程序设计者只需理解一些基本的概念，就可以用它编写出适用于各种情况的应用程序。Java丢弃了C++中很难理解的运算符重载、多重继承等概念；特别是Java语言使用引用代替指针，并提供了自动的垃圾回收机制，使程序员不必担忧内存管理。

2.面向对象	

&emsp;&emsp;Java语言提供了类、接口和继承等原语，只支持类之间的单继承，但支持接口之间的多继承，并支持类与接口之间的实现机制（关键字为implements）。Java语言全面支持动态绑定，而C++语言只对虚函数使用动态绑定。总之，Java语言是一个纯粹的面向对象程序设计语言。

3.安全性

&emsp;&emsp;Java安全可靠，例如，Java的存储分配模型可以防御恶意代码攻击。此外，Java没有指针，因此外界不能通过伪造指针指向存储器。更重要的是，Java编译器在编译程序时，不显示存储安排决策，程序员不能通过查看声明猜测出类的实际存储安排。Java程序中的存储是在运行时由Java解释程序决定。

4.跨平台性

&emsp;&emsp;Java通过JVM（虚拟机）以及字节码实现跨平台。Java程序由javac编译器编译成为字节码文件（.class）文件， JVM中的Java解释器会将.class文件翻译成所在平台上的机器码文件，执行对应的机器码文件就可以了。Java程序只要“一次编写，就可到处运行”。

5.支持多线程

&emsp;&emsp;Java语言支持多线程。所谓多线程可以简单理解为程序中多个任务可以并发执行，多线程可以在很大程度上提高程序的执行效率。

6.分布性

&emsp;&emsp;Java是分布式语言，既支持各种层次的网络连接，又可以通过Socket类支持可靠的流（stream）网络连接。用户可以产生分布式的客户机和服务器，在这个过程中，网络变成软件应用的分布式运载工具。

（1）Java语言是简单的

&emsp;&emsp;Java语言的语法与C语言和C++语言很接近，使得大多数程序员很容易学习和使用Java。另一方面，Java丢弃了C++中很少使用的、很难理解的、令人迷惑的那些特性，如操作符重载、多继承、自动的强制类型转换。特别地，Java语言不使用指针，并提供了自动的废料收集，使得程序员不必为内存管理而担忧。

（2）Java语言是面向对象的

&emsp;&emsp;Java语言提供类、接口和继承等原语，为了简单起见，只支持类之间的单继承，但支持接口之间的多继承，并支持类与接口之间的实现机制（关键字为implements）。Java语言全面支持动态绑定，而C++语言只对虚函数使用动态绑定。总之，Java语言是一个纯面向对象程序设计语言。

（3）Java语言是分布式的

&emsp;&emsp;Java语言支持Internet应用的开发，在基本的Java应用编程接口中有一个网络应用编程接口（java.net），它提供了用于网络应用编程的类库，包括URL、URLConnection、Socket、ServerSocket等。

（4）Java语言是健壮的

&emsp;&emsp;Java的强类型机制、异常处理、废料的自动收集等是Java程序健壮性的重要保证。对指针的丢弃是Java的明智选择。Java的安全检查机制使得Java更具健壮性。

（5）Java语言是跨平台的

&emsp;&emsp;Java语言具有跨平台性，且提出“一次编译、随处运行”的口号，这是因为Java源文件（.java）在Java平台上被编译为体系结构中立的字节码格式（.class），然后可以在实现这个Java平台（装有JVM）的任何系统中运行。这种途径适合于异构的网络环境和软件的分发。

（6）Java语言是可移植的

&emsp;&emsp;Java的可移植性源于其跨平台性，此外Java还严格规定了各个基本数据类型的长度，它们在不同的机型上都保持固定的长度。Java系统本身也具有很强的可移植性，Java编译器是用Java实现的，Java的运行环境是用ANSIC实现的。

（7）Java语言是多线程的

&emsp;&emsp;在Java语言中，线程是一种特殊的对象，它必须由Thread类或其子（孙）类来创建。Java语言支持多个线程的同时执行，并提供多线程之间的同步机制。

## Java语言的优势

### 1.使用广泛

&emsp;&emsp;Java是目前使用最广泛的网络编程语言之一,而使用广泛最直接的特点就是岗位多，就业率高并且薪资不俗。并且Java是大型企业中的一种流行选择，并且几十年来一直如此。它广泛用于构建企业级 Web 应用程序。众所周知，它非常稳定，因此许多大型企业都采用了它。

### 2.简单

&emsp;&emsp;Java的风格类似于C++，但摈弃了C++中容易引发程序错误的一些特性，如指针、结构、枚举以及内存管理等。同时Java提供了丰富的类库，可以帮助更方便的开发Java程序。

### 3.面向对象

&emsp;&emsp;面向对象这种基于对象的编程更符合人的思维模式，使人们更容易编写程序。

### 4.跨平台性

&emsp;&emsp;跨平台性是Java语言最大的优点。其他语言编写的程序都可能会因为操作系统的变化，处理器的不同以及核心资源的变化产生程序错误或者无法运行。Java的虚拟机完美的解决了这个问题，使用Java编写好的程序可以在任何安装了Java虚拟机的计算机上正确的运行，实现了"一次编译，处处运行"的编程目标。

### 5.多线程

&emsp;&emsp;Java内置了对多线程的支持。多线程允许同时完成多个任务。多线程使人产生了多个任务在同时执行的错觉，但实际上目前的计算机处理器在同一时刻只能执行一个线程，但处理器可以在不同的线程之间高速切换，由于处理器的速度远远超过了人接收信息的速度，所以给人的感觉好像多个任务在同时进行。
